> 基于设计先行的开发方法论，融合 Superpowers 理念的完整实践指南

---

## 概述

本指南展示如何使用这套配置完成一个完整的产品开发流程，从产品设计到部署上线。

**核心原则**: 设计先行 → 计划 → 执行 → 审查 → 完成

## 阶段一：产品设计

### 1.1 市场调研 → 头脑风暴

```
用户: "我想要做一个简单的 CRM 系统"
    ↓
AI: 检测到创造性请求，启动设计探索流程
    ↓
/brainstorm
    ↓
加载 brainstorming skill
    ↓
[设计探索对话]
```

**启动命令**:
```
/brainstorm
```

**对话示例**:
```
AI: 你好！我注意到你想要构建一个 CRM 系统。
    
    在我们开始写代码之前，让我先了解一些背景：
    
    1. 这个 CRM 系统的主要目标用户是谁？
    2. 核心功能有哪些？（如：客户管理、销售跟踪、任务管理等）
    3. 你希望用什么技术栈？
    4. 有没有类似的系统可以参考？
    
    请分享你的想法，我们一起把它转化为明确的设计规格。
```

### 1.2 需求分析 → 设计文档

通过 `/brainstorm` 对话，会生成设计规格文档：

```markdown
# CRM 系统设计规格

## 1. 项目概述
- 项目名称: Simple CRM
- 类型: Web 应用
- 核心功能: 客户管理、销售跟踪、任务管理
- 目标用户: 小型销售团队

## 2. 技术栈
- 前端: React + TypeScript + TailwindCSS
- 后端: Node.js + Express
- 数据库: PostgreSQL
- 认证: JWT

## 3. 核心功能模块

### 3.1 客户管理
- 客户列表（分页、搜索、筛选）
- 客户详情
- 新增/编辑/删除客户
- 客户分类标签

### 3.2 销售跟踪
- 销售机会管理
- 销售漏斗
- 成交记录

### 3.3 任务管理
- 任务列表
- 任务分配
- 截止日期提醒

## 4. API 设计

### 客户接口
- GET /api/customers - 客户列表
- GET /api/customers/:id - 客户详情
- POST /api/customers - 创建客户
- PUT /api/customers/:id - 更新客户
- DELETE /api/customers/:id - 删除客户

## 5. 数据库设计

### customers 表
- id, name, email, phone, company, tags, created_at, updated_at
```

### 1.3 设计批准

```

### 3.1 执行单个任务

```
用户: "确认"
    ↓
/execute
    ↓
加载 task-execution skill
    ↓
选择任务执行
    ↓
两阶段审查
```

**启动命令**:
```
/execute
```

**任务选择示例**:
```
AI: 请选择要执行的任务：

                    可执行任务
═══════════════════════════════════════════════════════════

## 最佳实践

1. **永远先设计** - 不管任务多小，都要先 `/brainstorm`
2. **计划驱动开发** - 使用 `/plan` 生成计划后再执行
3. **执行必审查** - 每个任务完成后都要经过两阶段审查
4. **定期评估质量** - 使用 `/quality` 保持代码质量
5. **测试驱动** - 复杂功能使用 `/tdd`
6. **完成必收尾** - 使用 `/finish` 正确完成分支
