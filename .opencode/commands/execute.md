# Execute Command - 任务执行 + 两阶段审查

## 核心理念

**两阶段审查**: 每个任务完成后必须经过两次审查，确保：
1. 实现符合设计规格
2. 代码质量达标

## 执行流程

### Phase 1: 准备

1. **检查是否有计划文档**
   - 读取 `docs/plans/` 目录下的计划文档
   - 如果没有计划文档，提示用户使用 `/plan` 命令

2. **解析任务**
   - 获取任务编号或任务描述
   - 提取任务内容、涉及文件、验收标准

### Phase 2: 激活 task-execution skill

**加载 skill**: `skill: task-execution`

task-execution skill 会：
1. 准备隔离上下文
2. 派发实现 agent
3. 执行两阶段审查

### Phase 3: 审查结果处理

**两阶段审查流程**:

```
实现完成
    ↓
阶段A: 规格合规审查 (spec-compliance-reviewer)
    ↓ 通过 / 不通过 ↓
阶段B: 代码质量审查 (code-reviewer)
    ↓ 通过 / 不通过 ↓
任务完成 / 修复后重审
```

**阶段A - 规格合规审查**:
- 实现是否符合设计规格？
- 是否有遗漏的功能点？
- 是否有超出设计的额外实现？
- 接口签名是否符合设计？

**阶段B - 代码质量审查**:
- 代码风格符合规范？
- 测试覆盖充分？
- 无安全漏洞？
- 无性能问题？

### Phase 4: 任务完成

1. **更新 TodoWrite**: 标记任务完成
2. **显示结果**: 告诉用户任务状态
3. **询问下一步**: 是否继续执行下一个任务

---

## 使用方式

```
/execute 1        → 执行任务1
/execute 2        → 执行任务2
/execute next     → 执行下一个未完成任务
/execute all      → 全部执行（需要多次确认）
```

---

## 禁止事项

| 反模式 | 正确做法 |
|--------|----------|
| 没有计划就执行 | 必须先使用 /plan 生成计划 |
| 跳过审查 | 两次审查都必须执行 |
| 同时执行多个任务 | 每次只执行一个任务 |
