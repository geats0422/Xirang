# Quality Command - 代码质量评估

## 概述

评估指定代码的质量，生成详细的评估报告，并根据评分决定是否需要优化。

## 使用方式

```
/quality <path>
```

## 参数

- `<path>`: 要评估的文件或目录路径（可选，默认当前目录）

## 评估流程

1. **收集信息**: 分析代码结构，运行静态检查
2. **维度评估**: 评估 6 个维度
3. **生成报告**: 输出详细评估报告到 `docs/reviews/YYYY-MM-DD-<feature>-quality.md`
4. **决策**: 根据评分决定是否优化

## 评估维度

| 维度 | 权重 |
|------|------|
| 代码可读性 | 20% |
| 代码复杂度 | 20% |
| 最佳实践 | 15% |
| 错误处理 | 15% |
| 性能 | 15% |
| 安全 | 15% |

## 评分等级

| 评分 | 等级 | 行动 |
|------|------|------|
| 90-100 | A | 优秀，无需优化 |
| 80-89 | B | 良好，可选优化 |
| 70-79 | C | 一般，建议优化 |
| 60-69 | D | 较差，需要优化 |
| <60 | F | 很差，必须优化 |
