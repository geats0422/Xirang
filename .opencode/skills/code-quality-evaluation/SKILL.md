# Code Quality Evaluation - 代码质量评估

## 概述

评估代码质量并生成详细的评估报告。根据评分决定是否需要优化。

**这是独立的代码质量评估流程**，不依赖于其他工作流。

## 评估维度

### 1. 代码可读性 (20%)

- 变量/函数命名是否清晰
- 是否有适当的注释
- 代码结构是否合理
- 是否有重复代码

### 2. 代码复杂度 (20%)

- 函数长度是否合理（<50 行）
- 文件大小是否过大（<800 行）
- 嵌套深度是否过深（>4 层）
- 圈复杂度

### 3. 最佳实践 (15%)

- 是否遵循语言惯例
- 是否使用标准模式
- 是否有统一的代码风格

### 4. 错误处理 (15%)

- 是否有适当的异常处理
- 是否有输入验证
- 错误消息是否友好

### 5. 性能 (15%)

- 算法复杂度是否合理
- 是否有不必要的重复计算
- 资源使用是否合理

### 6. 安全 (15%)

- 是否有硬编码凭据
- 是否有注入风险
- 是否有安全漏洞

## 评估流程

### Phase 1: 收集信息

1. **确定评估范围**
   - 如果指定路径，评估该路径
   - 如果未指定，询问用户

2. **运行静态分析**
   - Lint 检查
   - 类型检查
   - 复杂度分析

3. **读取关键文件**
   - 读取要评估的主要文件
   - 分析代码结构

### Phase 2: 维度评估

对每个维度进行评分：

```
评分标准：
- 90-100: 优秀，几乎没有问题
- 80-89: 良好，有少量小问题
- 70-79: 一般，有明显问题
- 60-69: 较差，有严重问题
- <60: 很差，有关键问题
```

### Phase 3: 生成报告

生成结构化报告，包含：

- 总体评分和等级
- 各维度评分
- 问题列表（按严重程度）
- 修复建议

**输出路径**: `docs/reviews/YYYY-MM-DD-<feature>-quality.md`

### Phase 4: 决策

根据评分与用户互动：

**评分 >= 80:**
```
评估完成。代码质量良好 (B级)。

[可选优化项]

是否需要进行优化？输入 "优化" 继续，或 "完成" 结束。
```

**评分 < 80:**
```
评估完成。代码质量需要提升 (C级或更低)。

发现 [N] 个关键问题，[M] 个警告。

是否需要我生成优化计划并执行优化？
输入 "优化" 开始优化，或 "完成" 结束。
```

## 前端代码特殊规则

如果是前端代码（React, Vue, Angular, Svelte 等），额外检查：

- 组件是否遵循组合优于继承
- 状态管理是否合理
- 是否有不必要的重新渲染
- 是否有性能问题（memoization, virtualization）
- 是否遵循框架最佳实践

使用 `frontend-patterns` skill 作为参考。

## 后端代码特殊规则

如果是后端代码（Node.js, Python, Go, Java 等），额外检查：

- API 设计是否遵循 REST 原则
- 是否有适当的中间件处理
- 数据库查询是否安全（防止 SQL 注入）
- 是否有适当的缓存策略
- 错误处理是否统一

使用 `backend-patterns` skill 作为参考。

## 数据库代码特殊规则

如果是数据库代码（SQL, PostgreSQL, MySQL 等），额外检查：

- 索引是否合理
- 查询是否优化
- 是否有适当的事务处理
- 迁移脚本是否安全
- 是否有适当的备份策略

使用 `database-migrations` 和 `postgres-patterns` skill 作为参考。

## 优化流程（如果用户选择优化）

1. **生成优化计划**
   - 按严重程度排序问题
   - 为每个问题制定修复方案
   - 估算工作量

2. **执行优化**
   - 按计划逐个修复问题
   - 每修复一个，验证是否有效

3. **重新评估**
   - 优化后重新评估
   - 确认评分提升

```markdown

- 评估范围: [path]
- 评估时间: [date]
- 报告路径: docs/reviews/YYYY-MM-DD-<feature>-quality.md

## 总体评分: [score]/100 ([grade])

## 各维度评分

| 维度 | 评分 | 等级 |
|------|------|------|
| 代码可读性 | XX | X |
| 代码复杂度 | XX | X |
| 最佳实践 | XX | X |
| 错误处理 | XX | X |
| 性能 | XX | X |
| 安全 | XX | X |

## 问题汇总

### 关键问题
1. [file:line] 问题描述
   - 严重程度: 关键
   - 修复建议: ...

### 警告
...

### 建议
...

## 结论
[总体评估]
```

## 注意事项

1. **必须有具体证据** - 每个评分必须有具体问题支持
2. **必须有位置** - 每个问题必须包含文件:行号
3. **必须有建议** - 每个问题必须有具体可执行的修复建议
4. **如果评分 >= 80** - 只提供可选优化，不强制
5. **如果评分 < 80** - 强烈建议优化，给出详细计划
