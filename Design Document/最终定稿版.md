# 最终定稿版

## 1. 账户与身份

### 主邮箱

- 每个账户仅有一个主邮箱
- 主邮箱用于：
  - 本地登录账号
  - 邮箱验证码接收
  - 找回密码
  - 安全通知

### 主邮箱迁移

- 允许将已绑定的第三方邮箱迁移为新的主邮箱
- 主邮箱切换后，后续验证码、找回密码、安全通知全部切换到新主邮箱

### 第三方绑定

支持：

- Google
- Microsoft
- GitHub

一个账户可以同时绑定多个第三方账号。
允许这些第三方账号的邮箱与主邮箱不一致。

------

## 2. 注册规则

### 邮箱注册

```text
输入邮箱
→ 设置产品密码
→ 邮箱验证码验证
→ 注册成功
```

### 第三方直接注册

规则：

```text
用户未先输入邮箱，直接使用第三方登录
→ 若第三方返回 verified 邮箱
→ 该邮箱可作为主邮箱创建账户
```

------

## 3. 登录规则

### 本地登录

```text
主邮箱 + 产品密码
→ 风险判断
→ 低风险：直接登录
→ 高风险：发送邮箱验证码
→ 登录成功
```

### 第三方登录

```text
Google / Microsoft / GitHub OAuth 登录
```

若满足以下条件，则自动合并到已有账户：

- 第三方返回 email
- `email_verified = true`
- 系统中存在相同邮箱账户（按邮箱比较规则处理后）

------

## 4. 密码规则

- 最小长度 ≥ 8
- 必须包含：
  - 大写字母
  - 小写字母
  - 数字
  - 特殊字符
- 禁止常见弱密码

这里还剩一个实现细节以后端决定：

- 弱密码黑名单范围
- 是否接入泄露密码库校验

这是实现层，不影响你现在的状态机。

------

## 5. 邮箱比较规则

采用轻标准化：

- 去首尾空格
- 转小写
- 不区分大小写
- 不合并 Gmail 点号
- 不合并 `+tag`

所以：

```text
Alice@gmail.com = alice@gmail.com
```

但：

```text
alice@gmail.com ≠ a.lice@gmail.com
alice@gmail.com ≠ alice+work@gmail.com
```

这意味着：

- `alice+work@gmail.com` 可以是独立账户
- `a.lice@gmail.com` 也可以是独立账户
- 自动合并只基于“标准化后完全一致”

------

## 6. 设备信誉规则

### 设备状态

- 新设备
- 历史设备
- 常用设备

### 成为常用设备

规则：

```text
7 天内连续 3 次成功登录
→ 成为常用设备
```

### 老化机制

规则：

```text
常用设备 30 天未使用
→ 降级为历史设备
```

### 老化后的处理

- 降级但不清零
- 不再享受常用设备免验证待遇
- 但保留历史信誉，不回到全新设备状态

------

## 7. IP 风险规则

- 使用 `/16` 网段作为判断粒度
- 若当前 IP 段不在最近 30 天记录中，则视为新来源

------

## 8. 登录验证码触发规则

### 发验证码

- 新设备登录
- 历史设备登录
- 常用设备登录，但 IP 段异常

### 不发验证码

仅满足以下条件时不发：

- 当前设备是常用设备
- 当前 IP 段在最近 30 天记录中

------

## 9. 敏感操作统一规则

以下场景都需要主邮箱验证码：

- 修改密码
- 绑定第三方
- 解绑第三方
- 更换主邮箱
- 删除账户

------

# 这套规则对应的最小状态机

## 设备状态机

```text
新设备
→ 7 天内连续 3 次成功登录
→ 常用设备
→ 30 天未使用
→ 历史设备
```

历史设备再次登录后：

- 需要验证码
- 后续若重新满足“7 天内连续 3 次成功登录”，再恢复为常用设备

------

## 登录判断矩阵

| 设备状态 | IP 是否异常 | 是否发验证码 |
| -------- | ----------- | ------------ |
| 新设备   | 否/是       | 是           |
| 历史设备 | 否/是       | 是           |
| 常用设备 | 否          | 否           |
| 常用设备 | 是          | 是           |

------

## 自动合并矩阵

| 场景                                            | 是否自动合并 |
| ----------------------------------------------- | ------------ |
| 第三方返回邮箱与已有主邮箱完全一致，且 verified | 是           |
| 仅大小写不同                                    | 是           |
| 仅点号不同                                      | 否           |
| 仅 `+tag` 不同                                  | 否           |
| 返回邮箱未 verified                             | 否           |

------
