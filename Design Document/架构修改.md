## C. 状态机设计（MVP/v1）

### 1) 路径版本状态机（`learning_path_versions.status`）

`pending -> generating -> ready`  
`pending/generating -> failed`  
`failed -> generating`（点击“重试生成路径”）

规则：
- 默认展示最新 `ready`
- `failed` 保留 7 天后清理
- 若无任何 `ready`，阻塞进入学习并提示失败+重试按钮
- 懒生成进入 mode 时触发
- 超时 120s 显示“仍在生成，可稍后刷新/重试”

### 2) 关卡进度状态机（`learning_path_progress.status`）

`locked -> unlocked -> completed`

规则：
- 线性解锁（完成即可解锁下一关）
- 每个 level = 一次 run 完成
- 各版本独立记录，不互相覆盖

### 3) 传奇复习状态机（单元维度）

`unit_completed -> legend_available -> legend_repeatable`

奖励倍率：
- 第1次 50%
- 第2次 30%
- 第3次 20%
- 第4次及以后 10%

附加规则：
- 免费用户每日封顶：XP300 / Coin60（UTC+8）
- 订阅用户（月/年）：不封顶
- 旧版本路径统一 70% 折扣（在上述倍率后或前固定顺序计算，后续在计划中锁）
