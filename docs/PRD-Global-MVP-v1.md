## 1. 本轮判断

- 当前阶段: P0 发布前的集成与验证阶段
- 核心问题: 不是再设计更多功能, 而是把“已经 A 级的前端壳”与后端、题库、经济系统、埋点、支付与评估闭环接起来
- 推荐方案: 保留三模式叙事, 但强制“共享题库、共享结算引擎、共享账本”, 本周只验证一个可重复学习循环

## 2. 北问题

> 用户有很多自己想学的资料, 但现有工具要么太被动, 要么太枯燥, 无法把私有资料转化为愿意每天回来完成的主动回忆。

## 3. 产品定义

### 3.1 一句话定位

Xirang 是一款面向全球的、内容无关的 UGC 游戏化学习引擎。用户上传任何学习资料, AI 自动拆解为可玩的题库与关卡, 再由三种不同心理节奏的模式承接学习循环。

### 3.2 新决策原则

任何功能都必须同时服务学习效果、全球可理解性和经济闭环, 否则延后。

### 3.3 当前错配

- 文档口径与前端实现仍有偏差
- 经济系统已经被想得很复杂, 但还没有真实激活/留存数据
- 真实项目路径里前端存在且可运行, 但后端与业务闭环尚未在当前仓库中被同等验证

## 4. 目标用户

### 4.1 产品层定位

- 产品是内容无关的, 不绑定 GitHub、考试、语言或某一个垂类
- 用户上传什么资料, 系统就基于什么资料生成题库

### 4.2 P0 发布切口

- 首发验证切口: 职业提升场景
- 首批资料类型: 用户个人笔记 + 课程讲义 PDF
- 原因: 资料私有性强、上传意愿更真实、付费动机更接近“节省时间 + 强制坚持”

### 4.3 次级验证人群

- 备考刷题党

### 4.4 现在不做什么

- 不做 GitHub 专用学习产品
- 不做企业协作知识库
- 不做 AI 播客/TTS
- 不做重 3D 特效
- 不做代币直接兑换真实 AI 算力

## 5. 目标与指标

### 5.1 North Star

- 每周有效学习关卡完成数

### 5.2 本轮唯一验证指标

- 激活率 >= 40%

### 5.3 激活定义

1. 注册后完成资料上传, 目标 80%
2. 上传后开始第一局, 目标 70%
3. 开始后完成第一局, 目标 75%

### 5.4 质量门槛

- 题目可上线准确率 >= 80%
- 解析失败率 <= 25%
- 每题支持一键反馈“这题有误”

### 5.5 风险指标

- <2 秒作答比例
- 余额不足购币触发率
- 连胜激冻使用率
- 同模式疲劳衰减触发率
- 用户标错率

## 6. 用户流程

1. 用户上传资料
2. UI 显示免费/BETA 免责声明
3. 后端生成统一题库 JSON
4. 进入模式选择页
5. 三模式平铺展示, 不做默认推荐
6. 用户选择模式进入关卡
7. 完成关卡后结算 XP / 代币 / 连胜状态
8. 用户继续下一关, 或去商城消费/购币

## 7. 模式系统

### 7.1 统一题型

- 单选
- 多选
- 判断
- 填空

### 7.2 题型配比

- 判断 30%
- 单选 35%
- 填空 25%
- 多选 10%

### 7.3 模式选择页策略

- 三模式平铺
- 不做默认推荐
- 选择辅助维度使用“爽感/压力”
- 标签:
  - 极速狂热 = 刺激
  - 无尽深渊 = 深度
  - 决战牌局 = 轻松

### 7.4 模式定义

#### 极速狂热
- 定位: 刺激
- 题型: 判断 + 短单选
- 交互: 滑动 / 快速判断 / 连击
- 价值: 最快建立心流, 吃掉碎片时间

#### 无尽深渊
- 定位: 深度
- 题型: 普通层单选/判断, Boss 层填空
- 交互: 红心机制 + Boss 关
- 价值: 高认知负荷, 高回报

#### 决战牌局
- 定位: 轻松
- 题型: 单选/多选/填空
- 交互: 卡牌化、拖拽化、Combo 化
- 价值: 降低移动端输入摩擦, 保留主动回忆

## 8. 经济系统

### 8.1 货币逻辑

- XP: 用于进度、排行、段位和成就感
- 代币: 用于购买道具

### 8.2 排行体系

- 统一 XP 大盘
- 周联赛与段位叙事
- 排位的核心目标是留存与身份感, 不是纯刷分

### 8.3 防刷机制

- 每题最短作答时间: 2 秒
- 快于阈值不计分
- 反馈语气: 游戏化调侃
- 同模式连续短时刷分触发收益递减

## 9. 道具与购币

### 9.1 已确认核心道具

#### 连胜激冻
- 用途: 保护断签
- 最大库存: 3 个
- 自动补充: 连续学习 15 天自动补 1 个
- 满仓处理: 额外补充资格转为少量 XP

#### XP Boost
- 用途: 短时间增加 XP 获取速度

### 9.2 购币规则

- 代币不足时允许现金购币
- 商城页常驻购币入口
- 点击购买道具且余额不足时弹出购币

### 9.3 购币结构

- 三档标准购币包: 小 / 中 / 大
- 最小购币包:
  - 定价: $1.99
  - 定位: 救急包
  - 目标场景: 刚好补一个连胜激冻

### 9.4 商业化边界

- P0 允许: 现金购币 + 代币买虚拟道具
- P0 不允许: 代币直接换真实 AI 算力
- P1 以后若要开放 AI 权益, 必须满足:
  - 连续 4 周激活率 >= 40%
  - D7 >= 20%
  - 周付费转化 >= 5%

## 10. AI 与评估闭环

### 10.1 P0 AI 范围

- 文档解析
- 题库生成
- 题目解析
- 错题来源片段回溯

### 10.2 最小评估集

- 任务完成率
- 关键错误率
- 用户可接受率
- 延迟
- 成本
- 用户标错率

### 10.3 闭环

1. 收集错误样本
2. 人工抽样错误分析
3. 调整 prompt / 规则
4. 回归固定样本集
5. 线上监控错误率变化
