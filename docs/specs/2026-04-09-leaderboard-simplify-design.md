# 排行榜功能简化设计规格

## 1. 背景与目标

### 1.1 问题描述
当前排行榜功能存在以下问题：
1. **好友排行榜未实现**：前端有"好友/全服"切换按钮，但后端尚未实现好友数据过滤
2. **用户体验困惑**：切换到好友榜显示的数据与全服榜完全相同
3. **不必要的UI复杂度**：未实现的功能按钮可能造成用户困惑

### 1.2 目标
简化排行榜功能，移除好友相关UI，只保留全服排行榜：
1. 移除前端好友/全服切换按钮
2. 确保排行榜始终显示全服数据
3. 减少不必要的UI复杂度

## 2. 技术方案

### 2.1 选定方案：简化UI方案
移除好友切换功能，只保留全服排行榜。理由：
- 好友系统尚未实现，保持切换按钮会造成用户困惑
- 简化后的UI更清晰，用户体验更好
- 为后续好友功能预留后端接口

### 2.2 不包含的内容
- 好友关系的建立和管理（后续版本）
- 好友数据过滤（后续版本）

## 3. 架构设计

### 3.1 数据流
```
用户访问排行榜 → 全服排行榜API → 前端页面
```

### 3.2 组件关系
```
DungeonScholarLeaderboardPage.vue
├── LeaderboardSummaryPanel.vue (用户摘要)
├── LeaderboardStandingsTable.vue (排行榜表格 - 移除scope切换)
└── API调用 → leaderboard.ts → 后端API (scope参数保留但忽略)
```

## 4. 接口设计

### 4.1 前端接口
- `getLeaderboardSnapshot(limit, offset)` - 获取排行榜快照（移除scope参数）
- API请求不再传递scope参数

### 4.2 后端接口
- `GET /api/v1/leaderboard` - 获取排行榜数据
- scope参数保留但始终返回全服数据
- 添加日志记录scope参数（后续功能预留）

## 5. 错误处理

### 5.1 前端UI处理
- 移除好友切换按钮，不显示错误
- 排行榜始终显示全服数据

### 5.2 后端处理
- 忽略scope参数，始终返回全服数据
- 记录日志便于后续调试

## 6. 测试策略

### 6.1 前端测试
- 测试排行榜页面加载
- 测试载入更多按钮功能
- 移除scope相关测试

### 6.2 后端测试
- 测试全服排行榜返回正确数据
- scope参数被忽略的场景

## 7. 风险与约束

### 7.1 风险
- 后续添加好友功能时需要重新修改UI

### 7.2 约束
- 保持现有技术栈：Vue 3 + FastAPI + PostgreSQL
- 不破坏现有全服排行榜功能
